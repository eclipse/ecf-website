 System Requirements

You will need the following already installed on your machine

  • JDK 1.5 - final release
    java version "1.5.0"
Java(TM) 2 Runtime Environment, Standard Edition (build 1.5.0-b64)
Java HotSpot(TM) Client VM (build 1.5.0-b64, mixed mode, sharing)
  • Eclipse 3.1 - M2 or M3
    Earlier versions may work, but we have not tested them.
 Downloads

 
Option 1 - Pollinate Update Site

  • Select Help > Software Updates > Find and Install...
  • In the Install/Update dialog, select Search for new features to install
  • On the next page, click the New Remote Site button
  • Enter Pollinate and http://download.eclipse.org/technology/pollinate/update-site in the New Update Site dialog
  • Expand Pollinate in the tree and check either Milestone Builds or Integration Builds (not both)
  • On the next page, check Eclipse Pollinate Tools and complete the Install
    ... The Pollinate install is > 6 Meg so have patience ... :-)
  • Problems? Check the update site FAQ entries.

Option 2 - Pollinate from ZIP

  • Download org.eclipse.pollinate- 1.0.0.M2.zip or org.eclipse.pollinate-1.0.0.I20041109.zip
  • Unzip both into your Eclipse installation directory (e.g. C:\eclipse)
  • Launch Eclipse (you may need to use the -clean command line switch or delete your configuration directory)
  • Problems? Check the zip and install FAQ entries.

Option 3 - Pollinate from CVS

  • Create a new workspace (recommended)
  • Define a new Java Classpath Variable named JAVA_HOME that points to your JDK 1.5 installation (e.g. C:\jdk1.5.0)
  • Change Java Compiler options to JDK 1.5 ("Compliance and Classfiles" tab of the Java > Compiler preference page)
  • Import the Pollinate team project set for anonymous access or Pollinate team project set for committers.
        (when using anonymous access, enter "anonymous" as the username, leave the password field empty, and check "save password")
  • Launch a runtime workbench to see Pollinate in action (be sure to include the -Xbootclasspath in the "VM Arguments" field of the configuration page for the run-time workbench similar to what is described above)
  • Note that GEF is now part of the development stack, hence the org.eclipse.gef and org.eclipse.draw2d projects in the team project sets. However, GEF also has on-line help which is not available through these projects. To obtain the on-line help, download GEF-SDK-3.0.1.zip, extract the org.eclipse.draw2d.doc.isv and org.eclipse.gef.doc.isv plug-ins into your Eclipse plug-in directory, and edit org.eclipse.gef.doc.isv/plugin.xml to remove the "org.eclipse.gef" prerequisite. After restarting Eclipse, you should see a "GEF Developer Guide" entry in the contents when you invoke Help>Help Contents. Alternatively, you can just install the entire GEF SDK, which will mean GEF is duplicated between your workspace and Eclipse installation, although this should not normally cause problems.
