eclipse committer gathering

Held July 27, 2005
 
On July 27, 2005 representatives from four Eclipse projects met informally to discuss cross-project dependencies, identify areas for cross-project coordinated work, and discuss general committer community maintenance issues.

When: July 27th, 2005, 12:30pm-4:30pm
Projects: Platform, Web Tools Platform, Test and Performance Tools Platform, Eclipse Communication Framework
Where: Open Technology Business Center, Beaverton Oregon

Attendees

ECF: Scott Lewis, Pete Mackie
Platform: Luc Bourlier, Jared Burns, John Wiegand
TPTP: Karla Callaghan, Sri Doddapaneni, Anurag Gupta, Guru Ngarajan, Tyler Thessian
WTP: Ted Bashor, Konstantin Komissarchik, Tim Wagner

Informal Meeting Agenda

  • Provide Comments/Needs Input to Committer Board Representatives
  • Cross-project Technical Fiscussion: Platform 3.2 Plan
 Notes: Discussion with Committer Board Reps
Committer Board Reps Present: Scott Lewis, John Wiegand
  • Projects need more ways to report process ideas/successes/failures to other projects
  • Need more/better ways for cross-project reporting of
    • API changes
    • Project plan changes
  • Email/Mailing Lists: Need to add cross-project dependencies
  • Need Wiki with development practices content (see bug #87949)
  • Automate cross-project change notifications for plan documents
  • Support/require better coordination of release timing across projects (see Release timeline)
  • Provide project dashboard (see bug #109224)
  • Web site improvement (emphasize/support project Phoenix)
  • Create/encourage opportunities for projects to share success stories, developed technologies, and best practices with other projects/committers
  • Bugzilla Improvement Ideas
    • Enhancement tracking (see bug #111005)
    • Enhance/detail committer guidelines for bugzilla usage
    • Detail conventions for use of 'priority' and 'severity' fields
    • TPTP project has created sets of custom queries: generalize and provide to all projects
  • Have committer input/reviews of EMO-provided infrastructure changes (see bug #104097...work request tracking system)
  • Provide basic committer community statistics: How many committers? Affiliation? By project? (see bug #106559)
  • Provide committer reputation system (bugzilla impl or not, whatevers easier)
  • Provide assistance for developers/contributors prior to becoming committers (see bug #105637)
    • More support systems for learning best practices, codebase
    • CVS lacking pserver proxy support for non-committers
    • How can we help contributors and developers ramp up to become full committers?
 Notes: Cross-Project Technical Issues
John Wiegand presented Platform 3.2 pre-release plans and received input/feedback from WTP, TPTP, and ECF committers

Areas for Greater Future Cross-Project Interaction

  • WTP -> Platform: Improved builder architecture (see bug # 37723)
  • WTP -> Platform: Support for hierarchical projects? More flexible workspaces. (see bug #106176)
  • ECF -> TPTP: ECF use of TPTP communications. ECF could use TPTP communications API as ECF provider.
  • TPTP -> ECF: DSDP, TPTP, ECF...commonalities? ECF could provide support for communications needs for several projects, including Device Software Development Platform
  • ECF -> WTP: Runtime access to DOM in embedded Browser or ECF-enabled shared web browsing
  • ECF > Platform/RCP: Support for namespaces and identity
 Attendee Observations about Meeting
