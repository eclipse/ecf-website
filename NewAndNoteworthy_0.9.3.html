ECF API Refactoring


Core Plugin Split into 3 Plugins


Previously, there was one core ECF plugin org.eclipse.ecf. This plugin was approximately 150K of code. In response to mailing list postings about ECF code size and applicability in multiple runtime environments we went about refactoring the ECF core plugins so that there was less code, and less code was required for extension APIs (like filetransfer, datashare, discovery, remoteservices).

There are now two core plugins: org.eclipse.ecf.identity and org.eclipse.ecf. org.eclipse.ecf depends upon org.eclipse.ecf.identity.

These are now the only plugins required for use of ECF, so the total code size for minimal ECF applications is 73K. All other ECF extension APIs depend on these two plugins org.eclipse.ecf.identity and org.eclipse.ecf. The org.eclipse.ecf plugin exports the packages and classes from org.eclipse.ecf.identity, so it is not necessary for the org.eclipse.ecf.identity plugin to be added to dependencies.
