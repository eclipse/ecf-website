Remote Service Admin

ECF 3.5 includes an full implementation of OSGi 4.2 Remote Service Admin (RSA), chapter 122 in the enterprise specification. The implementation aims to ease the replacement of discover and distribution systems with ECF's provider architechture. Through the support of the standardized OSGi endpoint description extender format (section 122.8), the new implementation obsoletes the old / RFC 119 file-based discovery.

XML-RPC Provider

ECF 3.5 includes a XML-RPC Provider. It implements the ECF Remote Services API (org.eclipse.ecf.remoteservice). The provider therefore adds the possibility to use a XML-RPC based infrastructure for the registration, distribution and use of remote services. Remote Service invocation through a proxy and/or async proxy is supported, too.



ECF4Felix

ECF4Felix relax the dependency between ECF and Equinox. Until the release of ECF 3.5 the Eclipse Communication Framework runs on Equinox only. ECF4Felix now adds a second option to run ECF: Apache Felix. Using the github-project ECF remote services are possible to run on Apache Felix with pax-runner.



ECF Documentation Project

ECF recently started the ECF Documentation Project. This project is an approach to improve the quality of the ECF documentation with the help of the community. It also aims to lower the entry barrier of ECF for new consumers and thus attract new consumers to use ECF.

Major Infrastructure Improvements

ECF received a donation by Remain Software to buy new hardware. The donation results in a new server hosted by OSU labs, which is a huge improvement of the ECF infrastructure. ECF was able to introduce Maven releases and nightly repositorys lately. Thus, the new hardware is used to improve consumers access to ECF on the one hand. On the other hand it allows to improve our continuous integration and testing infrastructure.
