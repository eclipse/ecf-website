Remote Control via XMPP

Remotely Open Eclipse View Using XMPP/IM


In New and Noteworthy v0.8.4 a new example application was introduced showing how to create new protocols using XMPP ("Application-Defined Peer-to-Peer Protocols Using XMPP"). Using this capability, it is now possible to remotely show and Eclipse view via the ECF Roster View

To remotely open an Eclipse view, select a buddy in the roster view and right click to open the context menu for that buddy.

Then select a view from the list

Then select OK and it will then be opened on the remote user's ECF client. If the remote user does not have an ECF client (i.e. using Google Talk, GAIM, or some other XMPP client, then the view open request will simply be ignored).
