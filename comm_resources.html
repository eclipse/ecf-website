 ECF Communication Resources Overview

You need something to communicate with when developing and deploying communication applications. We call these communication end-points. Typically your communication end-point is some form of a communication server to receive and manage your messages. Sometimes this end-point is nothing more than something to test your communication application development against.

In many cases the communication server does nothing more than forward your message to some other entity. Nowadays that other entity is frequently another user who you are interacting with real time. We call this IM (Instant Messaging). You’ve heard term often and probably are using IM today. Why not have IM and a simultaneous way to transfers files utilizing the same end-point? There are the kinds of capabilities we are working on today. We’re working both within the ECF APIs and the servers that provide this capability.

The ECF development team has written ECF based communication servers plus developed API access to other publicly available open source communication servers. These ECF servers are available as Java programs you can deploy yourself and in some cases the servers are, or soon will be, available for your use as a communication service.

Today our list of available communication servers is limited but must useful for current ECF based communication needs. Our list of communication end-point will grow noticeably in the future. Others servers are under development by the ECF committer team. Some communication server applications are waiting on IP licensing issues so we can make them available to you. (Our server applications frequently use the open source work of others. We expect waiting for IP licensing issues to resolve will sometimes be our release or deploy dilemma.)

Additionally, expect that other developers, utilizing the ECF APIs to provide communication applications, will develop and make available their communication servers. When appropriate and possible, we’ll make information available here about the deployment and use of communication servers developed by others organizations and development teams.

This communication resources page will grow over time. Here you will find the information on how to deploy ECF developed servers and those available from others. Furthermore the ECF project will make server services available for your ECF communication development utilization and production use.
 ECF Testing and Communication Servers

The ECF project maintains three servers for communication end-point testing and ECF based communication services. We refer to the servers by the names of:

Eclipse ECF: at Eclipse Foundation, Ottawa, Ontario - Canada

OSU OSL: at Oregon State University Open Source Lab, Corvallis, Oregon - USA

Intel DuPont: at Intel® Remote Access Services, DuPont, Washington - USA

We have an appreciation and special thanks to the above enterprises for hosting and managing the above servers for our ECF communication server needs. If you have an occasion to use these servers we ask that you too express your appreciation, given the opportunity.

We use these communication servers in a variety of ways. Some are open for semi-public communication services use while others are closed to they can be utilized for communication testing. The closed servers can be made available for testing use based on valid and meaningful proposed needs. In broad terms, we utilize the above three servers, in overlapping functionality, as follows:

Development

    ECF communication servers, where 100% uptime or full functionality is not guaranteed. Server purpose is software development, configuration establishment, and manual intervention protocol testing. Services may be down or partially functional for days at a time. Exclusive ECF committer access only, except for guest contributors where access and services status is managed by one or more ECF committers.

Comm Testing

    ECF communication servers, where near 100% services uptime and full functionality is expected. Short-term loss of services functionality for testing and evaluation purposes is allowed. Return to normal service operation will normally be a high-priority unless users are notified of the situation and expected service restoration. Exclusive committer access—other access exceptions may occur by written proposal plan and approved by the ECF project lead.

Build

    Sever(s) designated to run headless builds and code testing for Eclipse plug-in development. Build server is initially deployed for ECF plug-in build and code testing.

CVS

    SCM (Software Configuration Management) and source code archive for development not ready for general release mostly due to IP licensing issues--ECF release checkout and zip archive via scripting.

Production & Evaluation

    ECF communication server, where 100% services uptime and full functionality is guaranteed, except when short-term faults or anomalies occur. Loss of server functionality requires high priority restoration. Semi-public access allowed (login required) with limited administrative access by selected ECF team committer members.

If you have a communication end-point needs either for ECF development and testing or a production level communication server, send an email request to the ECF developers mailing list.

 ECF Server Services

Located on the ECF servers identified above at the ECF Testing and Communication Servers section the ECF project team maintains following ECF related communication services for ECF development at testing:

ECF Generic Collaboration Server

    The main ECF Generic server user features are private collaboration groups that are connected to resources in the Eclipse workspace, and within those groups: chat, peer file sharing, enhanced Eclipse presence info, URL sharing, and remote Eclipse view opening. The ECF Generic Collaboration Server also provides dynamically adding arbitrary new components (shared objects). This is a huge developer feature (as opposed to user feature).

Jive Messenger XMPP (Jabber) Server

    Jive Messenger is an Open Source Java based server for comprehensive group chat and instant messaging (IM), which is based on the open XMPP protocol. See Deploying the Jive Messenger Open Source XMPP (Jabber) Server for more information and downloads.

See the ECF New and Noteworthy page for information on the functional details the above servers provide and how to access via the Eclipse IDE.

At this time, public access to the above ECF servers is closed until we resolve the LDAP access log-in procedures. We are presently working on the logistics of connecting to a LDAP server for authenticated access to ECF servers. We’ll note here when this access is available.

If you have an ECF related experimental or evaluation need to access the above ECF communication services, send an email request to the ECF developers mailing list. You can also deploy your own ECF servers as discussed in the Installing and Configuring ECF Communication Servers for Development Testing section of this page.

    Connecting an ECF Generic Collaboration Client to a Test Server
    1. Make sure ECF plugins are installed in one of three ways described above.
    2. Open the Navigator View or the Package Explorer View.
    3. Select any Project (first create some new project if none previously exists).
    4. Right-click on the project to bring up the context menu for the selected project.
    5. Select the ECF menu toward the bottom of the menu and select one of:
      • 'Join Collaboration...' This will allow you to specify an URL of the group to join. You must know of a currently running server (and know it's URL) to use this option.
      • 'Join Test Collaboration (Localhost)' This will connect you directly to a test server running on localhost. See below for instructions for Running a Collaboration Server on Localhost if you wish to run this way.
      • 'Join Test Collaboration (Composent)' This will connect you directly to a test server running on composent.com. This is a test server setup for the ECF team usage. You are welcome to try out connecting to this server to test out/use the ECF example client. The server's availability may not be consistent, however.
    Running a ECF Generic Collaboration Server on Localhost
    1. Download ECF projects from CVS as described above.
    2. Choose Run... or Debug... from the Eclipse Run Menu.
    3. Under 'Java Application' choose 'ECF Collab Server'.
    4. Choose 'Run' (or Debug).
    5. This will run a simple server on your localhost machine with the ECF URL: ecftcp://localhost:3282/server. You can then connect to this server with an Eclipse client as described in Connecting an ECF Collaboration Client to an Existing Server above.
    Running the ECF Generic Collaboration Server Using the Server Feature
    1. Download ECF Generic Collaboration Server
    2. Unzip contents to desired install location <installdir>.
    3. Modify the config file appropriately located in <installdir>/features/org.eclipse.ecf.serverfeature_0.2.0/conf/server.xml
    4. Here is an example server.xml: 
      <server>
	<connector hostname="localhost" port="3282">
		<group name="server"/>
	</connector>
</server>
      This example configures a server that listens on port 3282, with hostname "localhost", and supporting a single group named "server". This corresponds to the following client URL: ecftcp://localhost:3282/server.

      Note that if the hostname attribute is omitted, the hostname will default to the fully qualified domain name of the given machine (e.g. www.eclipse.org).

      Note also that there can be as many groups as desired defined within a given port/connector, and each group will define a unique url. For example for group name="server1", or group name="server2", the URLS would be: ecftcp://localhost:3282/server1, ecftcp://localhost:3282/server2, respectively.
    5. Start the server by changing directory to <installdir>/features/org.eclipse.ecf.serverfeature_0.2.0/bin and running either startserver.sh or startserver.cmd/bat as appropriate to your OS.
